Valid Palindrome [LeetCode]

class Solution: def is_palindrome_convert_to_list(self, s: str) -> bool: strs = [] for c in s: if c.isalnum(): strs.append(c.lower()) while len(strs) > 1: if strs.pop(0) != strs.pop(): return False return True def is_palindrome_deque(self, s: str) -> bool: deque = collections.deque() for c in s: if c.isalnum(): deque.append(c.lower()) while len(deque) > 1: if deque.pop() != deque.popleft(): return False return True def is_palindrome_slicing(self, s: str) -> bool: s = s.lower() s = re.sub(r'[^a-z0-9]', '', s) return s == s[::-1]

8월 달 즘에 자바로 해결했었던 문제다.

그리 어려운 문제는 아니었으며 예제 코드가 길어보이는 이유는 3가지 해결 방안으로 문제를 해결했기 때문이다.

핵심 코드는 다음과 같다.

def is_palindrome_slicing(self, s: str) -> bool: s = s.lower() s = re.sub(r'[^a-z0-9]', '', s) return s == s[::-1]

파이썬의 문자열 슬라이싱을 이용하면 코드의 성능이 우월해진다.

슬라이싱 문법을 이용하면 문자열의 포인터 값이 바로 전달되기 때문이라고 한다.

Reorder Data in Log Files [LeetCode]

Reorder Data in Log Files

class Solution: def reorderLogFiles(self, logs: List[str]) -> List[str]: letters, digits = [], [] for log in logs: if log.split()[1].isdigit(): digits.append(log) else: letters.append(log) letters.sort(key=lambda x: (x.split()[1:], x.split()[0])) return letters + digits

파이썬에서 문자열 데이터를 조건으로 처리할 때 얼마나 직관적으로 처리하느냐에 관한 문제다.

파이썬 람다 키워드 사용방법이 신기했고, 다른 언어에도 있는 개념이라 어렵지 않게 습득했다.

리스트 끼리 + 연산할 수 있다는 것이 신선했다.

Group Anagrams [LeetCode]

Group Anagrams

class Solution: def groupAnagrams(self, strs: List[str]): # -> List[List[str]]: anagrams = collections.defaultdict(list) for word in strs: anagrams[''.join(sorted(word))].append(word) return list(anagrams.values())

같은 철자들이 들어간 단어끼리 리스트로 그룹화하는 문제다. 해시 테이블을 활용하면 간단하게 해결할 수 있는데 짧게 쓸 수 있어서 조금 벙쪘다.

anagrams = collections.defaultdict(list)

인자로 list 객체를 넣은 이유는 없는 값을 조회할 때 KeyError 발생을 방지하기 위함이다.
