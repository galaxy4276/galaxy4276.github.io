좋은 문제집을 소개받다.

스터디 원 중에 해당 정보를 확인하고 공유해준 충대 22년 SW-IT-Contest 토대로 일단 간단하게 3문제 진행해봤다.

많이 들어왔던 엘리트 충대라 첫 문제부터 각오하고 들어갔는데 생각보다 난이도는 괜찮았다.

사진 자료 상 2문제는 너무 쉬웠으므로 간단한 스택문제인 "균형 잡힌 소떡소떡" 만 짚어가겠다.

25641. 균형 잡힌 소떡소떡 [BOJ]

균형 잡힌 소떡소떡

using namespace std; int main() { ios::sync_with_stdio(false); cin.tie(nullptr); stack<char> S; int t_count = 0, s_count = 0; string T; cin >> T; string ST; cin >> ST; for (int i = ST.length() - 1; i >= 0; i--) { char c = ST[i]; if (c == 't') t_count++; else s_count++; S.push(ST[i]); } while (t_count != s_count) { if (S.top() == 't') t_count--; else s_count--; S.pop(); } while (!S.empty()) { cout << S.top(); S.pop(); } cout << endl; }

정말 반가운 스택문제다.

나는 스택문제가 좋다 ㅎㅎ

처음에 좌측 재료를 뺄 때 소시지(S)를 빼야하는데 떡(T) 이 있으면 떡을 빼놓고,

소시지(S) 빠지면 뺀 떡(T)을 다시 밀어넣어야하는 줄 알고 괴상한 로직을 작성했다 ㅎㅎ;

자신감 채우는 하루였다.

날로먹었어?

스터디 조건은 만족했지만 이걸 알고리즘 공부했다고 하긴 민망하다.

사실 우리 스터디는 여유롭게 가면서 노 커리큘럼 체계긴 하다.

게으르게 한 주를 보낸 건 아니다.

급작스럽게 스프링 개발을 총괄담당하게 되었고, 홈쇼핑 프로젝트 프런트엔드 작업하느라 바빴다.
